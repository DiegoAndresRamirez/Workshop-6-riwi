1 question: Porque se ejecuta asi el codigo? Podemos apreciar que en nuestro codigo hay 2 setTimeOut, estos elementos son considerados por JS como tareas asincronas, lo que significa que tienen que esperar a que ocurra cualquier evento para que se ejecuten, en este caso los segundos que le ponemos al final;

JavaScript ejecuta su codigo de la siguiente manera: primero busca todas las tareas sincronas, es decir, las que no tienen ningun bloqueo ni condicion para ejecutarse, y se ejecutan de arriba hacia abajo, despues que se ejecuten todas las tareas sincronas, empieza a buscar las tareas asincronas, que ya las explique, ya aqui entra la condicion setTimeOut, le pasamos un parametro de milisegundos, y el codigo evalua cual funcion tiene que esperar mas tiempo para ejecutarse, y finalmente ordena el que tenga menos bloqueos o en este caso segundos y los ordena de menor a mayor...

Como conclusion, se ejecuta primero el console.log() que esta al principio del codigo, aunque lo podemos poner en cualquier lugar, luego se ejecuta el primer setTimeOut() ya que tiene menos segundos por esperar, y finalmente se ejecuta el segundo setTimeOut()... me canse

Ejercicio 10 : Visualización del Event Loop

¡Hola de nuevo Usuario Cibernetico!, ahora podras ver desde otro punto de vista como se ejecuta el Event Loop, debajo de este texto hay un boton, si lo presionas copiaras un codigo de ejemplo y serás redirigido a otra pagina donde podras ver el event loop en todo su esplendor, cuando quieras!

console.log("Inicio del script"); // Macrotarea: setTimeout setTimeout(() => { console.log("Macrotarea 1 second (setTimeout)"); }, 1000); setTimeout(() => { console.log("Macrotarea 0 seconds (setTimeout)"); }, 0); // Microtarea: Promesa Promise.resolve() .then(() => { setTimeout(() => { console.log("Macrotarea (setTimeout) inside Microtarea 1"); return "from micro 1"; }, 0); }) .then((message) => { console.log("Microtarea 2 (Promesa)"); }); // Microtarea: Promesa Promise.resolve() .then(() => { console.log("Microtarea 3 (Promesa)"); }) .then(() => { console.log("Microtarea 4 (Promesa)"); }); console.log("Fin del script");

Si lo lograste, te daras cuenta que a pesar de que en el codigo que copiaste habia un orden especifico, el resultado fue totalmente diferente, eso es debido al Event Loop, verás... el event loop es la manera en la que JS lee las tareas en un codigo y la forma en la que los ejecuta; Hay tres puntos a tener claros:

1: Macrotareas : son tareas asíncronas que se ejecutan después de que el event loop ha completado la fase actual, en el código dado, las macrotareas son los temporizadores setTimeout() con un tiempo de espera de 1000ms y 0ms, estas tareas se agregan a la cola de macrotareas y se ejecutan después de que se completan todas las microtareas

2: Microtareas : son tareas asíncronas que se ejecutan después de que se resuelvan las promesas y antes de que se ejecuten las macrotareas en la siguiente iteración del event loop, en el código dado, las microtareas son las funciones then() asociadas con las promesas resueltas, estas tareas se agregan a la cola de microtareas y se ejecutan antes de las macrotareas en la siguiente iteración del event loop.

En resumen, las macrotareas se ejecutan después de que se completan todas las microtareas, y las microtareas se ejecutan antes de las macrotareas en cada iteración del event loop.
